> [!note] Enunciado
> 1. Mostrar que Textbook RSA para firma es inseguro (ataque de no mensaje) usando $m = \sigma^e \mod N$.
> 2. ¿Por qué se reduce el riesgo de este ataque si se usa Hash RSA?

## 1. Ataque de no mensaje sobre Textbook RSA

En Textbook RSA, la firma de un mensaje $m$ es $\sigma = m^d \mod N$, y la verificación comprueba que $\sigma^e \equiv m \pmod{N}$.

**Ataque:**

1. Dada la clave pública $p_k = \langle N, e \rangle$, el atacante elige un valor $\sigma$ cualquiera.
2. Calcula $m = \sigma^e \mod N$.
3. Emite el par $\langle m, \sigma \rangle$ como si fuera un mensaje firmado.

El par **siempre verifica**, ya que la verificación calcula:

$$\sigma^e \mod N = (\sigma)^e \mod N = m$$

El atacante no necesita conocer la clave privada $d$ para producir un par válido $(m, \sigma)$; simplemente elige la firma primero y deriva el mensaje a partir de ella. El problema es que puede generar firmas válidas para mensajes que no eligió libremente, pero suficiente para quebrar la seguridad del esquema.

## 2. Por qué Hash RSA mitiga el ataque

En Hash RSA la firma se calcula como:

$$\sigma = [hash(m)]^d \mod N$$

y la verificación comprueba que $hash(m) \equiv \sigma^e \pmod{N}$.

Si el atacante intenta el mismo ataque: elige $\sigma$, calcula $m = \sigma^e \mod N$ y emite $\langle m, \sigma \rangle$, la verificación ahora compara $hash(m)$ con $\sigma^e \mod N$, y solo valida si:

$$hash(m) = \sigma^e \mod N$$

Para que esto se cumpla, el atacante necesitaría encontrar un $m$ tal que $hash(m) = \sigma^e \mod N$, lo cual requiere invertir la función hash o encontrar una colisión. Si $hash$ es resistente a preimagen y a colisiones, esto es computacionalmente inviable.
